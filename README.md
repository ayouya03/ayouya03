## 📂 Projets Phares

### 📈 [Real-time Sentiment Analysis of the Stock Market](https://github.com/ayouya03/Real-time-Sentiment-Analysis-of-the-Stock-Market)
* **Technologies :** Apache Kafka, Apache Spark, MongoDB, LLM, Docker, Git
* **Description :** Conception et déploiement d'un pipeline complet d'analyse de données en temps réel. Le projet croise les flux de données boursières en direct avec une analyse des sentiments (via LLM) d'articles financiers afin de fournir des indicateurs d'aide à la décision pour les investissements boursiers.

### ✈️ [Plateforme Data en temps réel et en batch]([https://github.com/ayouya03](https://github.com/ayouya03/Real-Time-Batch-Data-Platform))
* **Technologies :** Apache Spark, Python, Kafka, Airflow, Snowflake, Docker, Kubernetes, MySQL, Elasticsearch
* **Description :** Une plateforme de données entièrement conteneurisée avec ingestion par lots, streaming en temps réel, un entrepôt de données en schéma en étoile, le suivi des expérimentations de machine learning, une API REST .NET 8 et une observabilité complète — le tout orchestré à travers 20 services Docker gérés par une seule stack Docker Compose.

### 🏅 [Real-Time-Blood-Pressure-Monitoring-System]([https://github.com/ayouya03](https://github.com/ayouya03/Real-time-Analysis-of-blood-pressure-data)) 
* **Technologies :** Kafka · Python · FHIR · Elasticsearch · Kibana · Machine Learning
* **Description :** Ce projet implémente un pipeline de streaming temps réel pour la surveillance de la pression artérielle de patients. Les données médicales sont produites au format FHIR, transmises via Kafka, analysées en temps réel, puis stockées selon leur criticité. Les cas anormaux sont indexés dans Elasticsearch pour analyse et visualisation dans Kibana, tandis que les cas normaux sont archivés localement.
